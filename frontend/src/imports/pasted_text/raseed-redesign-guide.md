Improve the current Raseed — رصيد design without removing any existing pages or features.

Main issue:
The current design has problems with colors, visual hierarchy, spacing, layout organization, and overall SaaS polish. Redesign the UI to feel more professional, clean, trustworthy, and ready to sell to real supermarket and retail store owners.

Important:

* Keep the product name: Raseed — رصيد.
* Keep Arabic RTL.
* Keep all pages and features already created.
* Do not remove POS, Dashboard, Products, Inventory, Reports, Users, Settings, SaaS Admin Panel, Onboarding, Shift Management, End-of-Day Closing, Subscription, and Support pages.
* Improve only the visual design, layout structure, organization, spacing, components, and usability.

Brand direction:
Raseed should feel:

* Professional
* Trustworthy
* Modern
* Simple
* Retail-focused
* Financially reliable
* Easy for supermarket owners and cashiers

Color system:
Create a better professional color palette.

Use:

* Primary color: a strong modern blue or teal suitable for SaaS and trust.
* Secondary color: a calm green for sales/profit/success.
* Neutral background: soft light gray, not pure white everywhere.
* Cards: clean white cards with subtle borders and shadows.
* Danger: red only for refunds, delete, stock danger, and critical alerts.
* Warning: amber/orange only for low stock, expiry, and attention alerts.
* Success: green only for profit, completed sales, active status, and paid invoices.
* Info: blue for general actions and system information.

Avoid:

* Too many random colors.
* Childish gradients.
* Overly saturated colors.
* Dark backgrounds on main business pages.
* Inconsistent button colors.
* Using red/green/blue randomly without meaning.

Layout improvements:

* Make the sidebar cleaner and more organized.
* Group navigation items logically:

  1. Operations:
     الكاشير، الشيفتات، إغلاق اليوم
  2. Store Management:
     المنتجات، التصنيفات، المخزون، الموردين، أوامر الشراء
  3. Finance:
     المبيعات والفواتير، المرتجعات، المصاريف، العملاء والديون
  4. Insights:
     التقارير، التنبيهات، الذكاء والتحليلات
  5. Admin:
     المستخدمين والصلاحيات، سجل النشاط، الاشتراك والفواتير، الإعدادات
  6. Help:
     المساعدة والدعم
* Add clear active state in sidebar.
* Make icons consistent in size and style.
* Make the top bar cleaner with:
  Store name, branch selector, online/offline status, notifications, and user menu.

Dashboard improvements:

* Make dashboard cards consistent in size.
* Use clear numbers and labels.
* Add small comparison indicators like “+12% عن أمس”.
* Improve spacing between cards.
* Make charts cleaner and easier to read.
* Add quick actions in a clean horizontal row.
* Do not overcrowd the dashboard.

POS screen improvements:
The POS page must be the most practical page.

* Make the barcode/product search input very large and obvious.
* Make cart and payment area visually separated.
* Use large action buttons.
* Keep total/final amount very clear.
* Use a simple 2-column or 3-column layout suitable for cashier work.
* Make colors functional:
  Complete sale = primary/success.
  Cancel = neutral/danger.
  Hold order = warning.
  Refund = danger.
* Avoid dashboard-style clutter on POS.
* Make it look fast and touch-screen friendly.

Tables:
Improve all tables:

* Better spacing.
* Clear headers.
* Search bar above tables.
* Filters grouped neatly.
* Status badges with consistent colors.
* Row actions in a clean dropdown or action buttons.
* Pagination design.
* Empty states when no data exists.

Forms and modals:

* Make forms cleaner.
* Use grouped sections.
* Use clear labels.
* Add validation states.
* Add helpful placeholder text.
* Use modals for simple add/edit actions.
* Use full pages for complex forms like product details, purchase order, and settings.

Typography:

* Use a clean Arabic-friendly font style.
* Clear hierarchy:
  Page title
  Section title
  Card title
  Body text
  Helper text
* Make numbers in dashboards and POS large and readable.
* Avoid tiny text in important operational screens.

Design system:
Create and apply a consistent design system:

* Buttons:
  Primary, secondary, ghost, danger, success.
* Inputs:
  Default, focus, error, disabled.
* Cards:
  Dashboard card, alert card, report card, product card.
* Badges:
  Active, inactive, paid, refunded, low stock, expired, pending.
* Alerts:
  Success, warning, error, info.
* Modals:
  Confirmation, add/edit, details.
* Empty states:
  No products, no sales, no reports, no notifications.
* Loading skeletons.
* Toast notifications.

Page organization:
For every page:

* Add a clear page header.
* Add short page description when useful.
* Put primary action button in the top area.
* Put search and filters under the header.
* Keep content organized into cards/sections.
* Avoid random spacing.
* Avoid inconsistent card sizes.
* Avoid overcrowding.

SaaS Admin Panel improvements:
Make it feel separate but still part of Raseed.

* Cleaner platform dashboard.
* Better subscription status colors.
* Clear stores table.
* Better plan cards.
* Clear MRR and active stores widgets.
* Professional support tickets layout.

Mobile/tablet responsiveness:

* Owner dashboard should look good on tablet/mobile.
* POS can remain desktop-first.
* Tables should adapt with cards or horizontal scroll.
* Sidebar should collapse on smaller screens.

Final goal:
Make the Raseed design look like a polished, premium SaaS product, not a rough prototype. Keep the Arabic RTL interface, keep all existing functionality, but completely improve the colors, spacing, visual hierarchy, navigation organization, component consistency, and overall professional feel.
